# Homepage Wireframe

This document translates the homepage blueprint into a practical wireframe and content outline.

It is not final copy.
It is a structure guide for design and implementation.

---

## Homepage Model

Narrative flow:

1. Arrival
2. Belief
3. Authority
4. Proof
5. Insight
6. Conversion

---

## Section 1 — Hero

### Purpose

Make a strong first impression.
This section should make people remember Wahyu before they fully understand every detail.

### Role in narrative

`Arrival`

### Content priority

- one dominant statement
- one short support line
- one primary CTA
- one secondary CTA or small link

### Wireframe

````text
------------------------------------------------------
| minimal nav                                         |
|                                                     |
| small label / positioning                           |
| BIG statement headline                              |
| short support line                                  |
|                                                     |
| [primary CTA]   [secondary CTA]                     |
|                                                     |
| animated visual / portrait / motion-led element     |
------------------------------------------------------

### Suggested content type

- label:
  - Strategic SEO consultant
  - Technical SEO specialist
  - SEO Growth Mindset

- headline style:
  - sharp, memorable, statement-led
  - should feel more like a personal thesis than a resume summary

- support line:
  - 1 short sentence only

### Copy rule

Do not explain too much here.
Hero should create attraction, not full understanding.

### Motion rule
- this is the strongest motion section
- motion can lead here
- should feel premium and intentional
- should not become chaotic or gimmicky

---

## Section 2 — Manifesto / Point Of View

### Purpose

Explain how Wahyu sees SEO in a way that feels personal and sharp.

### Role in narrative

`Belief`

### Content priority

- 2 to 4 short statements
- optionally one short intro line

### Wireframe

```text
------------------------------------------------------
| section label                                       |
| short editorial intro                               |
|                                                     |
| statement 1                                         |
| statement 2                                         |
| statement 3                                         |
------------------------------------------------------
````

### Suggested content type

- SEO is not just rankings, but structure
- SEO should make websites clearer, not noisier
- growth comes from systems, not random tactics
- technical depth matters when it changes outcomes

### Copy rule

- statements should be sharp
- avoid paragraph-heavy explanation
- think manifesto, not service description

### Motion rule

- staggered reveal is enough
- typography pacing matters more than effects

---

## Section 3 — Authority / Method

### Purpose

Ground the brand with credibility.
This section should answer: “Is there real depth behind the statement?”

### Role in narrative

`Authority`

### Content priority

- 1 short intro
- 3 method pillars or trust signals
- maybe one compact credibility line

### Wireframe

```text
------------------------------------------------------
| section label                                       |
| short authority intro                               |
|                                                     |
| pillar 1      pillar 2      pillar 3                |
|                                                     |
| optional supporting line or micro-proof             |
------------------------------------------------------
```

### Suggested pillar themes

- Technical depth
- Systems thinking
- Structured execution

### Alternative angle

Instead of generic pillars, this could become:

- How I think
- How I diagnose
- How I build

### Copy rule

- concise
- intelligent
- no resume-style wall of text

### Motion rule

- subtle transitions
- strong layout and typography should carry the section

---

## Section 4 — Selected Work

### Purpose

Show proof without turning the homepage into a full portfolio archive.

### Role in narrative

`Proof`

### Content priority

- 2 or 3 selected projects only
- each with:
  - title
  - one-line framing
  - one result / outcome / type of impact

### Wireframe

```text
------------------------------------------------------
| section label                                       |
| selected work heading                               |
| short framing line                                  |
|                                                     |
| featured project                                    |
| smaller project                                     |
| smaller project                                     |
|                                                     |
| [view all projects]                                 |
------------------------------------------------------
```

### Suggested structure

- one dominant featured project
- two secondary projects

This avoids a flat, repetitive grid.

### Copy rule

- enough proof to create trust
- not full case-study detail
- details belong on projects page

### Motion rule

- hover and transition behavior can be more expressive here
- should reinforce premium feel

---

## Section 5 — Writing / Insight

### Purpose

Show that Wahyu has a perspective, not just execution skill.

### Role in narrative

`Insight`

### Content priority

- one short heading
- 1 to 3 writing teasers max
- one link to blog / insights

### Wireframe

```text
------------------------------------------------------
| section label                                       |
| insight / writing heading                           |
| short support line                                  |
|                                                     |
| article teaser 1                                    |
| article teaser 2                                    |
| or one strong featured article                      |
|                                                     |
| [read insights]                                     |
------------------------------------------------------
```

### Copy rule

- do not make this a big blog index
- use it as an authority signal

### Motion rule

- restrained
- should feel editorial

---

## Section 6 — CTA / Contact

### Purpose

Give a clean next step.

### Role in narrative

`Conversion`

### Content priority

- one closing statement
- one primary CTA
- optional secondary contact path

### Wireframe

```text
------------------------------------------------------
| closing line / final statement                      |
| short supporting line                               |
|                                                     |
| [start a conversation]                              |
| optional email / contact link                       |
------------------------------------------------------
```

### Copy rule

- confident
- calm
- not pushy

---

## Suggested Homepage Hierarchy

This is the recommended content weight from top to bottom:

### Highest emphasis

- hero
- manifesto

### Medium emphasis

- selected work
- authority

### Lower emphasis

- writing
- CTA

This means:

- the homepage should be remembered for the first-screen feel and point of view
- not for how many sections it contains

---

## Suggested Content Density

### Hero

- very light

### Manifesto

- light

### Authority

- light to balanced

### Work

- balanced

### Writing

- light

### CTA

- very light

---

## Suggested Design Rhythm

The visual pacing should feel like:

1. strong arrival
2. pause
3. statement
4. controlled proof
5. quiet close

Not:

1. hero
2. another card grid
3. another card grid
4. another text block
5. CTA

---

## What To Remove From Current Homepage

Likely remove or reduce:

- generic explanatory cards
- repeated credibility wording
- too many descriptive paragraphs
- sections that say similar things in different wording

---

## What To Preserve

Keep these ideas, but restructure them:

- strong personal positioning
- technical SEO credibility
- structured thinking
- selected proof
- writing as authority

---

## Implementation Note

Before redesigning the homepage, each section should be written as:

- section goal
- max word count
- visual priority
- motion priority

That will prevent the homepage from becoming text-heavy again.
